macOS natively understands a narrow slice of the media landscape — primarily MPEG audio and video inside MPEG containers. If you work with MKV files, WebM video, AVI archives, Flash recordings, or Windows Media content, Finder shows blank icons, QuickLook shows nothing, and Spotlight can't search any of the metadata. QuickLook Video closes that gap. Install it once, and all three systems — Finder, QuickLook, and Spotlight — gain support for the formats that macOS leaves out.

## What QuickLook Video adds

QuickLook Video installs three system extensions that work silently in the background. You never interact with them directly; they hook into macOS and take effect automatically for any supported file you open or browse.

## Supported file formats

QuickLook Video adds support for the following container formats:

| Format | Extensions |
|--------|------------|
| Matroska | `.mkv`, `.mka` |
| WebM | `.webm` |
| Windows Media | `.wmv`, `.wma` |
| Ogg Video | `.ogv`, `.ogm` |
| AVI | `.avi` (partial support) |
| Flash Video | `.flv`, `.f4v` |
| Real Media | `.rm`, `.rv`, `.ra` |
| SMPTE | `.mxf`, `.gxf` |

## How it works

QuickLook Video installs three extensions into macOS:

- **Spotlight mdimporter** — reads metadata from non-native files and writes it to the Spotlight index so you can search by title, codec, resolution, duration, and more.
- **FormatReader Media Extension** — teaches AVFoundation how to demux non-native container formats and audio codecs, enabling QuickLook previews and Finder thumbnails.
- **VideoDecoder Media Extension** — adds software decoding for non-native video codecs that AVFoundation does not support natively.

All three extensions are bundled inside **QuickLook Video.app** and registered automatically when you first launch it. You do not need to manage them manually.

<Note>
Long-time Mac users may recognise this approach from [Perian](https://www.perian.org), which performed a similar function with the QuickTime framework before Apple replaced it with AVFoundation. QuickLook Video is the spiritual successor to Perian for the modern macOS extension model.
</Note>

## System requirements

- **macOS 12 (Monterey) or later**
- **Late-2013 Mac or newer** — the FFmpeg libraries that power codec support require AVX2 CPU instructions. Macs from late 2013 onward include AVX2. Earlier hardware is not supported.
