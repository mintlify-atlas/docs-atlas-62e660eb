QuickLook Video is an open-source macOS app licensed under GPL v2. You can build it yourself from the [GitHub repository](https://github.com/Marginal/QuickLookVideo). The build requires Xcode, a handful of Homebrew tools for the FFmpeg dependency chain, and initialized Git submodules. The steps below walk you through the full process.

## Prerequisites

Before you open Xcode, make sure you have the following:

- **Xcode** with the **macOS 12 SDK** or later. Download Xcode from the Mac App Store or the [Apple Developer portal](https://developer.apple.com/xcode/).
- **Homebrew** for installing the FFmpeg build tools. If you do not have Homebrew, see [brew.sh](https://brew.sh).

## Build steps

<Steps>
  <Step title="Install build tools">
    FFmpeg and its dependencies (dav1d, zimg) require Meson, Ninja, pkg-config, and the nasm assembler. Install them via Homebrew:

    ```bash
    brew install meson ninja pkg-config nasm
    ```
  </Step>

  <Step title="Clone the repository and initialize submodules">
    After cloning, initialize the Git submodules. The FFmpeg, dav1d, and zimg source trees are included as submodules and must be present before building.

    ```bash
    git submodule update --init --recursive
    ```
  </Step>

  <Step title="Open the Xcode project">
    Open `QLVideo.xcodeproj` in Xcode:

    ```bash
    open QLVideo.xcodeproj
    ```
  </Step>

  <Step title="Build the QuickLook Video target">
    In Xcode, select the **QuickLook Video** scheme from the scheme selector and press **⌘B** (or choose **Product → Build**). This builds all products, including the app, plugins, and FFmpeg libraries.
  </Step>
</Steps>

## Products

Building the **QuickLook Video** target produces the following:

| Product | Description |
|---|---|
| **QuickLook Video.app** | The main application. Hosts plugins and extensions, and registers Uniform Type Identifiers for the media types that the plugins handle. |
| **mdimporter** | Spotlight plugin that provides metadata for non-native file formats. |
| **formatreader** | AVFoundation Media Extension that adds support for non-native file formats and audio codecs. |
| **videodecoder** | AVFoundation Media Extension that adds support for non-native video codecs. |
| **simpleplayer** | Debug app for testing `formatreader` and `videodecoder`. Plays files using AVFoundation and exercises both extensions. |
| **ffmpeg** | FFmpeg libraries that the plugins depend on. The build also produces a standalone `ffprobe` executable for use in bug reports. |
| **dav1d** | AV1 codec library. A dependency of FFmpeg. |
| **zimg** | Format and colour conversion library. A dependency of FFmpeg. |

<Note>
  The `previewer` and `thumbnailer` QuickLook app extensions are listed in the Xcode project but are **not included in v3** of the app. Previews and thumbnails are now handled by the AVFoundation Media Extensions (`formatreader` and `videodecoder`).
</Note>

## FFmpeg dependency scripts

If you need to rebuild the native libraries independently, the repository includes dedicated build scripts:

```bash
scripts/buildffmpeg   # Build FFmpeg libraries
scripts/builddav1d    # Build the dav1d AV1 codec library
scripts/buildzimg     # Build the zimg conversion library
```

In normal development you do not need to run these manually — the Xcode build system invokes them as needed.

## Crash handling in Release builds

FFmpeg's demuxers and codecs can crash on corrupt or incompletely downloaded media files. In **Release** builds, both `formatreader` and `videodecoder` install exception handlers that silently kill the worker process when a crash occurs, preventing any crash dialog from appearing to the user.

In **Debug** builds, these exception handlers are not active, so crashes propagate normally. This makes Debug builds more useful for catching and diagnosing issues during development.

For guidance on attaching a debugger to the extensions, see the [Debugging](/debugging) page.
